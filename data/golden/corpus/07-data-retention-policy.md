# Data Retention Policy

## Retention Periods by Data Type

- **Application logs** are retained for **90 days**, then permanently deleted.
- **Backups** are retained on a rolling **35-day** window.
- **User data** retention depends on its classification tier under the Security Policy's Data Classification section: Restricted user data (customer PII and payment data) is retained only for as long as specified in the relevant customer contract, with no fixed company-wide default. Internal and Confidential operational data is retained for **1 year** after it's no longer actively used.

## Deletion Requests

A verified user request to delete their data must be fully processed within **30 days**.

## Legal Holds

If Legal issues a hold on specific data, that data must be preserved regardless of any retention period above until the hold is formally lifted by Legal.

## Data Classification Cross-Reference

This document's retention rules assume familiarity with the four classification tiers (Public, Internal, Confidential, Restricted) defined in the Security Policy's Data Classification section — refer there first if a tier's meaning is unclear.
